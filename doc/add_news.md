# 動線仕様（起動→ニュース追加まで最短化）

## 目的
起動〜WebView到達~再生までの基本動線を規定する

---

# HomePage
- 「＋ 新規作成」→ 2. ニュースセット新規作成モーダルが表示される
- ニュースセット名が自動で埋まっている(日付-連番)入力欄と、下の選択肢がセレクトボックスで選べる
    - 検索して追加 -> Google検索へ遷移
    - ニュースから追加 -> Googleニュースへ遷移
    - その他のサイトから追加 -> URL入力欄が現れ、入力されたURLへ遷移
- 次へボタンで、WebViewPageへ移動 (Google検索、Googleニュース、任意URL)

# WebViewPage
- Flutter側で表示するヘッダー、ヘッダーにはニュースセット名(鉛筆マークから変更可能)、WebView(画面のほぼ全体を覆う)、一括設定ボタン(WebViewのZ位置的な上、XY的な右下1)、ニュース追加モードトグルボタン(右下2。右下1の下)、戻るボタン(左下。及びヘッダの左側)。
- ニュース追加モードをオンにしてリンククリックすると「リンク先をニュースセットに追加しました」「取消」がSnackbarで表示 (3秒)。
- 「取消」を押すと取り消せる
- 一括設定ボタンを押すと「以下のリンク先をニュースセットに追加します」と聞かれる。プレビュー的に、リンク先のページのmeta name="description、ない場合はtitleタグ、それもない場合は最初のh1タグ、それもない場合URLの先頭100文字をプレビューとして表示する。
- 「OK」を押すとキューにリンク先の内容が追加される (SQLite3で内容が保存される)
- 「戻る」を押すとホーム画面に戻る。

# HomePage ~ NewsSetDetailPage
- ホーム画面に戻ると、ニュースセットの一覧に先ほど追加したニュースセットが表示されている。
- ホーム画面でも、一覧からボタン押下で再生はできる
- ニュースセットを押すとニュースセット詳細画面にいく。
- 詳細画面からは、再生、ニュースセット名変更、ニュース追加 (WebViewPageに移動)、ニュース削除ができる。
