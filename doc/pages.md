# 画面一覧

## 1. ホーム（ニュースセット一覧 / キュー一覧）
- 役割：保存済みの「ニュースセット（＝再生キュー）」を一覧表示
- 主な要素
  - ニュースセット一覧（タイトル / 件数 / 最終更新日時）
  - 「＋ 新規作成」ボタン
  - （任意）検索：セット名で絞り込み
- 主な操作
  - セットをタップ → 2. ニュースセット詳細へ
  - 「＋ 新規作成」→ 2. ニュースセット詳細（新規）
    - ニュースセット名が自動で埋まっている(日付-連番)入力欄と、下の選択肢がセレクトボックスで選べる
      - 検索して追加 -> Google検索へ遷移
      - ニュースから追加 -> Googleニュースへ遷移
      - その他のサイトから追加 -> URL入力欄が現れ、入力されたURLへ遷移
    - 次へボタンで、WebViewPageへ移動 (Google検索、Googleニュース、任意URL)

## 2. ニュースセット詳細（セット編集・キュー管理）
- 役割：セット名編集、記事キューの確認、再生開始、追加導線
- 主な要素
  - セット名（編集可能）
  - 記事リスト（タイトル / ドメイン / 追加日時）
  - 「再生」ボタン（連続再生開始）
  - 「記事を追加」ボタン
- 主な操作
  - 記事をタップ → 5. 記事プレビュー（任意）へ
  - 記事をスワイプ削除（任意）
  - 「再生」→ 3. プレイヤーへ
  - 「記事を追加」→ 4. WebView（記事追加）へ

## 3. プレイヤー（連続読み上げ）
- 役割：ニュースセット内の記事を順に読み上げ（TTS）
- 主な要素
  - 再生中のタイトル / 出典 / 進捗（何件目/総件数）
  - 再生/一時停止
  - 次へ / 前へ
  - スピード調整（任意）
  - 「元記事を開く」（任意）
- 主な操作
  - 「元記事を開く」→ 4. WebView（閲覧専用 or 追加モードOFF）へ
  - 戻る → 2. ニュースセット詳細へ

## 4. WebView（記事追加）
- 役割：Web検索・閲覧しながら、リンクを“1操作”でキューに追加
- 主な要素
  - WebView
  - 右上トグル「追加モード ON/OFF」（デフォルトOFF）
  - （任意）URLバー / 戻る・進む・リロード
- 挙動
  - 追加モードOFF：通常のブラウズ（リンク遷移OK）
  - 追加モードON：リンクタップ＝「遷移キャンセル」＋「キューに追加」
    - 追加後はSnackBar「キューに追加しました」＋「もとに戻す」
- 主な操作
  - 戻る → 2. ニュースセット詳細へ

## 5. 記事プレビュー（任意）
- 役割：キューに保存された記事の情報確認（全文・要約・整形後テキストなど）
- 主な要素
  - タイトル / 出典 / 追加日時
  - 整形済みテキスト（任意）
  - 「元記事を開く」
- 主な操作
  - 「元記事を開く」→ 4. WebViewへ
  - 戻る → 2. ニュースセット詳細へ

## 6. 設定（任意）
- 役割：読み上げ・保存・WebViewの設定
- 例
  - 読み上げ速度、声、句読点の扱い
  - 追加モードのデフォルトON/OFF
  - URL正規化（utm削除等）のON/OFF
  - データ削除（全セット削除）
  - プライバシーポリシー表示


# 画面遷移（フロー）

## 新規でニュースセットを作って聴く
ホーム
  →（＋新規作成）→ ニュースセット詳細（新規）
    →（記事を追加）→ WebView（記事追加）
      →（追加モードONでリンクタップ）→ キューに追加（SnackBar Undo）
      →（戻る）→ ニュースセット詳細
        →（再生）→ プレイヤー
          →（戻る）→ ニュースセット詳細
            →（戻る）→ ホーム

## 既存セットを聴く
ホーム
  →（セット選択）→ ニュースセット詳細
    →（再生）→ プレイヤー

## 既存セットに記事を足す
ホーム
  → ニュースセット詳細
    → WebView（記事追加）
      → キューに追加（Undo可）
      → ニュースセット詳細


# 認証/ログイン
- 不要（完全ローカル完結）
- データはSQLiteに保存（端末内のみ）
